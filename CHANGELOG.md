# Changelog

## 1.1.32

### 重点更新
- 探索「更多影视」页：首屏仍快速返回榜单，未覆盖条目通过后台分批查询 Emby/飞牛在库状态，在库角标异步补齐；预取时同步合并角标映射。

## 1.1.31

### 重点更新
- 修复 TG Bot 启动时因 Telegram API 超时导致整个应用无法启动的问题；Bot 连接失败时主服务照常运行，可在设置中稍后重启 Bot。

## 1.1.30

### 重点更新
- 订阅自动转存：首个分享链接失效时，自动尝试下一条链接并补充搜索，直至转存成功或无可试资源。
- 刷新页面时采用非阻塞登录校验，移除长时间「正在验证登录状态」全屏等待。
- 修复 TG Bot「保存配置」504 超时；优化探索「更多」页首屏加载与卡片布局稳定性。
- 精简 115 扫码登录可选设备列表。

## 1.1.29

### 重点更新
- 修复订阅检查任务因查询未包含 `douban_id` 导致启动时报错的问题。
- 容器启动时不再立即执行订阅检查与榜单同步，避免阻塞 `/health` 就绪检测。

## 1.1.28

### 重点更新
- 移除探索首页启动预热：减轻容器启动时的外部 API 压力，首页榜单仍按视口懒加载。

## 1.1.27

### 重点更新
- 修复 TG Bot「保存配置」按钮长时间 loading：保存与重启分离，并为 Bot 启停增加超时。
- 运行时设置保存请求延长超时，避免配置写入被误判为无响应。

## 1.1.26

### 重点更新
- 修复探索页搜索后左侧导航无法跳转其它页面的问题（搜索 URL 改走 Vue Router）。
- 优化侧栏菜单导航逻辑，避免探索子菜单与路由冲突。
- 界面设置保存：延长运行时配置超时并改进错误提示与 loading 状态。

## 1.1.25

### 重点更新
- 豆瓣/TMDB「更多影视」探索页：触底预取与展示逻辑优化，滚动加载更顺滑。
- 豆瓣探索分页请求降低同步 TMDB 解析条数，减轻分页接口延迟。

## 1.1.3

### 重点更新
- 影视探索首页海报骨架屏改为按当前容器宽度动态计算数量，避免宽屏下始终只显示固定 6 张占位卡片。

## 1.1.2

### 重点更新
- 探索页首屏改为优先返回豆瓣列表，TMDB 匹配改后台异步回填，降低 NAS 重启后的首屏阻塞。
- Nginx 与前端不再把探索页真实 502 误提示为“后端启动中”，便于定位外部源或代理问题。
- 容器启动日志补充前后端成功提示，并延长健康检查启动宽限时间。

## 1.1.1

### 重点更新
- 新增可选 Kafka 埋点能力，用于记录搜索、订阅、转存等事件。
- 提升订阅定时调度稳定性，补充重启恢复、误触发容错和重复执行保护。
- 优化容器启动阶段体验，前端可更早可用，后端预热期间不再直接暴露原始 502。
- TG Bot 影视通知和交互消息新增海报预览。
- 打通转存后的自动归档和归档完成后的自动 STRM 生成链路。

### 后端
- 新增 `media_postprocess_service`，统一处理转存后的归档和 STRM 自动化流程。
- 修复 STRM 设置保存校验，允许先保存配置，再在真正生成时检查运行前提。
- 增强调度器手动执行能力，支持禁用任务检查和 `force` 解锁卡住任务。
- 将 `misfire_grace_time` 调整为更稳妥的策略，避免服务重启后错过应执行任务。
- 统一订阅后台执行和定时执行的并发保护，减少同频道重复跑任务的风险。

### 前端
- 新增后端启动中的友好提示与自动等待逻辑，降低启动期 502 干扰。
- 同步更新 STRM 页面与启动流程相关的交互体验。

### 媒体流程
- 手动转存、探索页转存、工作流转存、订阅自动转存统一接入：
  `保存到 115 -> 自动归档 -> 自动生成 STRM`
- 离线下载完成后仍按原有监控链路自动触发归档，并在归档完成后继续自动生成 STRM。

### Telegram Bot
- 影视详情、资源列表、转存结果、离线结果、订阅通知均支持海报预览。

### Docker
- `compose.yaml` 增加可选 `KAFKA_BOOTSTRAP_SERVERS` 透传。
- all-in-one Nginx/API 启动流程增加后端预热兜底逻辑，提升启动阶段可用性。
